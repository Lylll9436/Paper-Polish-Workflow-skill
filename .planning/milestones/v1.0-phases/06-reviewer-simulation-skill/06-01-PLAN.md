- Use Structured Interaction when available; fall back to plain-text questions otherwise.

**Workflow -- Read-Analyze-Report (Single-Phase):**

Step 1 -- Collect Context:
- Load expression-patterns.md overview.
- If target journal specified, load journal template from `references/journals/[journal].md`. If template missing, refuse with message: "Journal template for [X] not found. Available: CEUS."
- Read user input: file via Read tool, or pasted text from conversation.
- If input is partial (only introduction, only methods, etc.), refuse with message: "This Skill requires the full paper for review. Please provide the complete manuscript."

Step 2 -- Analyze Paper:
- Read the full paper to understand claims, evidence structure, and writing quality.
- Load relevant expression pattern leaves based on paper sections encountered (introduction-and-gap.md for intro, methods-and-data.md for methods, etc.) to inform Writing Quality dimension assessment.
- Assess across all five dimensions:
  1. **Novelty** -- Is the contribution genuinely new? Incremental or transformative?
  2. **Methodology** -- Is the experimental design sound? Are baselines adequate? Ablation studies present?
  3. **Writing Quality** -- Expression clarity, academic register, AI-sounding patterns, terminology consistency
  4. **Presentation** -- Figure/table quality, section organization, flow between sections
  5. **Significance** -- Does the work matter? Practical or theoretical impact?
- If journal template loaded, cross-reference journal-specific expectations (e.g., CEUS requires geography/urban-systems perspective) and note where the paper falls short of journal fit.
- IMPORTANT: Generate concerns FIRST, then derive dimension scores from the concerns. This prevents score-concern inconsistency (Research pitfall 2).

Step 3 -- Generate Review Report:
- Write the review report in the following structure (locked decision):

```markdown
# Peer Review Report

**Paper:** [title or filename]
**Target Journal:** [journal name or "General"]
**Date:** [date]

## Scoring Overview

| Dimension | Score (1-10) | Justification |
|-----------|:---:|---------------|
| Novelty | X | [one-line justification] |
| Methodology | X | [one-line justification] |
| Writing Quality | X | [one-line justification] |
| Presentation | X | [one-line justification] |
| Significance | X | [one-line justification] |

## Major Concerns

### Major Concern N: [Descriptive Title]

**Section:** [section name, e.g., "Results (Section 4)"]

**Problem:** [description of the issue]

**Why this matters:** [impact on the paper's quality or publishability]

**Suggestion:** [one-sentence directional guidance, e.g., "Consider adding comparison with baseline X"]

> **[Chinese]** [Chinese translation of all three parts -- preserve domain terminology precisely]

## Minor Concerns

### Minor Concern N: [Descriptive Title]

[Same three-part structure as Major Concerns, with inline Chinese translation]

## Questions for Authors

1. [Question with enough context for the author to understand what is being asked]
   > **[Chinese]** [Chinese translation]

## Verdict

**Recommendation:** [Accept / Minor Revision / Major Revision / Reject]

[One-sentence summary of the overall assessment]

> **[Chinese]** [Chinese translation of recommendation and summary]
```

Step 4 -- Output:
- **File input:** Write the review report to `{input_filename_without_ext}_review.md` in the same directory as the input file. If input filename is unclear, use `review-report.md`.
- **Pasted text input:** Present the complete review report directly in conversation (no file to write to).
- Report the total number of major concerns, minor concerns, and questions identified.

**Chinese Translation Guidelines (Claude's Discretion -- inline blockquote chosen per research recommendation):**
- Use inline blockquote format (`> **[Chinese]** ...`) immediately after each concern's English text.
- Preserve domain terminology in Chinese (e.g., "ablation study" as "消融实验", not a generic paraphrase).
- The Chinese translation covers all three parts (problem, why, suggestion) in a single blockquote.

**Output Contract:**
| Output | Format | Condition |
- Review report | Markdown file or conversation output | Always |
- Scoring overview | 5-dimension table with 1-10 scores | Always |
- Concern count | Summary line | Always reported |

**Edge Cases:**
| Situation | Handling |
- Input is partial (only one section) | Refuse: "This Skill requires the full paper for review." |
- Input is too short (< 1 page / < 300 words) | Warn: "Text appears too short for a full paper review. Provide the complete manuscript." |
- Input language is Chinese | Warn and suggest Translation Skill first; proceed with review if user confirms |
- Journal template missing when journal specified | Refuse: "Journal template for [X] not found. Available: CEUS." |
- No significant weaknesses found | Still produce the report with high scores; note strengths in each dimension rather than fabricating concerns |
- Paper is clearly outside journal scope | Flag as a Major Concern about journal fit |

**Fallbacks:**
| Scenario | Fallback |
- Structured Interaction unavailable | Ask 1-2 plain-text questions (journal + input source) |
- Expression pattern reference missing | Proceed with general writing quality assessment; warn user |
- Journal template missing (no journal specified) | Ask once; if declined, review with general academic criteria |
- Write tool fails for output file | Present report in conversation instead |
