**Purpose** — One paragraph: translates Chinese academic drafts to polished English LaTeX for journal submission, with bilingual comparison. Mention glossary support and journal-aware style.

**Trigger** — List activation conditions and example invocations (English + Chinese).

**Modes** — Table with all 4 modes. Default: `direct` (per user decision: single-pass, no multi-round). In `interactive` mode, confirm section detection and glossary application before translating. In `batch` mode, process multiple sections with same settings.

**References** — Table for required references (expression-patterns.md) and leaf hints. Include loading rules: load journal template when specified, load geography-domain.md when spatial content detected, load anti-ai-patterns.md for vocabulary screening.

**Reference Selection Logic** — Sub-section under References. Encode the section-detection heuristic from RESEARCH:
1. Scan input for Chinese section keywords: 引言, 研究方法/方法, 数据, 结果, 讨论, 结论
2. Clear match -> load corresponding leaf module directly
3. Ambiguous -> load overview and use Quick Picks
4. Geography/urban/spatial content -> also load geography-domain.md
5. When target journal specified -> load its template alongside

**Ask Strategy** — Three pre-questions per RESEARCH:
1. Target journal (if not specified)
2. Custom glossary file path (if user has domain-specific terms)
3. Which section this text belongs to (only if ambiguous from content)
Rules: In direct mode skip when context sufficient. If specified journal has no template at `references/journals/[journal].md`, refuse translation and instruct user to add the template first. Never ask more than 3 questions.

**Workflow** — Four steps per RESEARCH workflow outline:
- Step 1 Collect Context: ask pre-questions, load references, load journal template (or refuse if missing), load glossary, select expression pattern leaf module
- Step 2 Translate: single-pass translation; apply journal style; use expression patterns; prioritize glossary terms; re-translate all text uniformly including inline English fragments; preserve ALL LaTeX commands verbatim ($...$, $$...$$, \begin{}, \end{}, \cite{}, \ref{}, \label{}, custom commands); avoid anti-AI vocabulary patterns
- Step 3 Generate Output: write `[name]_en.tex` (English-only with header comment showing source file, target journal, date); write `[name]_bilingual.tex` (Chinese paragraphs as LaTeX % comments, English as body text, paragraph-numbered); for pasted text use `translation_en.tex` / `translation_bilingual.tex` in cwd; preserve original input file unchanged
- Step 4 Self-Check: report terminology decisions (especially glossary applications), flag uncertain translations with explanation, note structural changes (sentence splitting/reordering), warn about potential issues (long sentences, missing refs)

**Output Contract** — Table: english_tex (always), bilingual_tex (always), self_check_summary (always, in session not in files).

**Edge Cases** — Table covering at minimum:
- Input too short (single sentence) -> translate but warn limited context
- No section headers in input -> ask user or default to overview patterns
- Mixed Chinese-English text -> re-translate everything uniformly per user decision
- LaTeX-heavy input (mostly formulas) -> preserve formulas, translate surrounding text
- Pasted text with no filename -> use fallback naming convention
- Journal specified but no template -> refuse, instruct user to add template
- Glossary file not found at specified path -> warn and proceed without glossary

**Fallbacks** — Table:
- Structured interaction unavailable -> 1-3 plain-text questions
- Expression pattern leaf missing -> use overview entrypoint
- Journal template missing -> refuse per locked decision (NOT proceed with general style — this is a locked decision override of the skeleton default)
- Anti-AI patterns file missing -> proceed without, rely on journal style preferences

**Examples** (recommended section) — One concise example showing: user provides a file path + specifies CEUS journal, Skill loads references, translates, produces two output files, appends self-check.

**Line budget compliance:** Target 250-280 lines. Do NOT inline expression patterns, journal rules, or anti-AI vocabulary. Keep workflow steps as concise bullet lists. Avoid paragraphs where bullets suffice.
