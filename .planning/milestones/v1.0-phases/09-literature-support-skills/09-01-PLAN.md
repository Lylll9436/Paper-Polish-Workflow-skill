**## Purpose**
One paragraph: Skill searches academic literature using the Semantic Scholar MCP, presents an interactive numbered result list (up to 10 papers), waits for the user to select a specific paper, then generates a verified BibTeX entry built exclusively from MCP-returned data. Before any workflow begins, a pre-flight probe confirms MCP availability — if unavailable, the Skill refuses with setup instructions. After the user selects a paper, a mandatory anti-hallucination verification prompt reminds the user to confirm the paper's relevance before committing the citation. This Skill does not write or rewrite paper content; it searches, selects, and generates citation entries only.

**## Trigger**
Activates when user asks to search literature, find papers, or generate BibTeX/citations. Include 6 bilingual examples (English + Chinese) covering: find papers on a topic, search literature for a keyword, generate BibTeX for a citation, 帮我找文献, 检索相关论文, 帮我生成引用格式.

**## Modes**
Table with two rows:
- direct (default): single-shot search → result list → user selects → BibTeX generated; no iteration
- batch: NOT supported — each search requires user selection in session

**## References**
Required table: none (this is a search task, not a writing task — no expression pattern files needed).
Leaf hints: none.
Note: "No reference files loaded. All paper metadata sourced from Semantic Scholar MCP at runtime."

**## Ask Strategy**
The Skill collects search inputs conversationally. Two questions at most:
1. Search query: the topic or keyword to search (required — ask if not in trigger)
2. Result count preference: how many results to display (optional — default: 5 results; max: 10)

If user provides the topic in the trigger message, skip question 1 and proceed immediately.

**## Workflow**
Five steps:

Step 1 — MCP Pre-flight Check:
- Call mcp__semantic-scholar__papers-search-basic with {"query": "test", "limit": 1}
- If call succeeds: proceed to Step 2
- If call fails or tool unavailable: refuse immediately with the exact error message from the interfaces block above. Do NOT proceed.

Step 2 — Collect Search Query:
- Extract search topic from trigger if present; otherwise ask (Ask Strategy question 1)
- Determine result count (default: 5)

Step 3 — Execute Search and Build Result Cards:
- Call mcp__semantic-scholar__papers-search-basic with the user's query and result count
- For each returned paper: call mcp__semantic-scholar__get-paper-abstract to fetch abstract excerpt (if abstract field is empty in search result)
- Display numbered result cards using the locked format from the interfaces block:
  **[N] Title**
  Authors: ... · Year: ... · Citations: ...
  > "One-sentence abstract excerpt" (or "Abstract not available" if absent)
- End the display with: "Please select a paper (enter number), or type 'none' to cancel."

Step 4 — User Selection:
- Wait for user to enter a number or "none"
- If "none": acknowledge and end session
- If valid number: confirm selection by restating the paper title
- Display the MANDATORY verification prompt: "Please confirm that the selected paper actually supports the claim you are citing — do not rely solely on the title."

Step 5 — Generate BibTeX:
- Construct BibTeX entry using ONLY the fields returned by the MCP for the selected paper
- Use the locked BibTeX format from the interfaces block
- Citation key format: first_author_lastnameLowercaseYYYYfirstKeyword (e.g., smith2023urban)
- If any field (volume, pages, DOI) is not returned by MCP: omit that field from the entry — never fill from prior knowledge
- Display the BibTeX block in a code fence

**## Output Contract**
Table with output type:
- bibtex_entry: BibTeX @article{} (or @inproceedings{}, @misc{} as returned by MCP type field) block in conversation; all fields populated from MCP data only

Include one concrete example showing a complete BibTeX entry with the five core fields (title, author, journal/booktitle, year, doi). Note: "All field values must come from MCP-returned paper record. Never supply missing fields from prior knowledge — omit them."

**## Edge Cases**
Table covering:
- MCP pre-flight fails → refuse with setup instructions (exact message from interfaces block); never proceed to search
- Search returns zero results → inform user: "No papers found for '[query]'. Try broader keywords or re-trigger with different terms."
- Abstract field missing for a result → show "Abstract not available" in result card; do not skip the paper from results
- User selects a number outside the displayed range → ask to re-enter a valid number
- User types "none" → acknowledge ("Search cancelled.") and end session
- MCP returns paper type that is not @article (e.g., conference paper, preprint) → use @inproceedings or @misc as appropriate; never force @article
- DOI not returned by MCP → omit doi field from BibTeX; add a comment: % DOI not available — please verify manually

**## Fallbacks**
Table covering:
- Semantic Scholar MCP unavailable → refuse immediately with setup guidance (Step 1 handles this; no partial fallback)
- mcp__semantic-scholar__get-paper-abstract call fails for one paper → mark abstract as "Abstract not available" and continue displaying other results
- Structured Interaction unavailable → ask search query as plain-text question; proceed with numbered list display in conversation
- mcp__semantic-scholar__paper-search-advanced available if basic search returns consistently poor results → mention in fallback: "If results are poor, try re-triggering with more specific keywords"

**## Examples**
One complete invocation example:
- User: "Find papers about urban heat island mitigation"
- Skill: [pre-flight passes silently] → displays 5 result cards → user selects [2] → verification prompt shown → BibTeX generated from MCP data
Show the result card format and the final BibTeX block in the example.

Line budget: aim for 230–270 lines. The MCP tool names and BibTeX format are the largest sections — keep workflow steps as numbered lists, not prose paragraphs.
