Write all required sections per skill-conventions.md:

**## Purpose**
One paragraph: Skill accepts experiment result data (tables, statistics, or result descriptions) and runs a two-phase workflow. Phase 1 extracts measurable findings and presents them for user confirmation. Phase 2 generates discussion paragraphs with grounded evidence + interpretation language, connecting to user-provided literature or placeholders. Never invents literature connections.

**## Trigger**
Activates when user asks to analyze experiment results, generate discussion paragraphs, or write discussion from result tables. Include bilingual examples (English + Chinese).

**## Modes**
Table:
- direct (default): full two-phase workflow (Phase 1 → user confirm → Phase 2)
- batch: NOT supported — experiment analysis requires full context of the results set

**## References**
Required table: references/expression-patterns.md (always).
Leaf hints table:
- results-and-discussion.md: loaded in Phase 2 for result reporting and pattern interpretation language
- conclusions-and-claims.md: loaded in Phase 2 for calibrated claim language (suggests, indicates, scope boundaries)
- methods-and-data.md: loaded if user's result description includes method details requiring clarification
Conditional: when user specifies a journal, load references/journals/[journal].md; if missing, refuse.
Anti-AI: load references/anti-ai-patterns/vocabulary.md in Phase 2 to avoid AI-sounding output.

**## Ask Strategy**
Ask before starting:
1. Research questions: "What are the main research questions this experiment addresses?" (required — Phase 2 needs these to connect findings)
2. Prior work to connect to (optional — ask once; if declined, use [CONNECT TO: ...] placeholders in Phase 2)
3. Target journal if not specified (ask once; if declined, use general academic style)
Rules: never ask more than 3 questions before starting; research questions are mandatory input.

**## Workflow**
Structure as two clearly labeled phases:

**### Phase 1: Analyze Results**

Step 1 — Prepare:
- Load expression-patterns.md overview
- If journal specified, load journal template; if missing, refuse
- Read input: file via Read tool, or pasted results block (table, statistics, narrative), or structured_data input
- Guard — measurable data required: if input is vague ("my results show improvement" without values), refuse with "Please provide specific values, comparisons, or metrics before I can identify findings."

Step 2 — Extract Findings:
- Identify measurable comparisons: method A vs. method B, magnitude, direction
- Identify trends: performance across conditions, dataset sizes, subgroups
- Identify outliers: results that deviate from the overall pattern
- For each finding, require: a direction (better/worse/higher/lower), a magnitude or value, and a comparison group or condition

Step 3 — Present Finding List:
- Use locked format per item: "Finding N: [subject] [comparison/trend] [value] on [metric/condition]"
- Summary line: "Identified N findings. Please confirm, correct, or add before I write discussion."
- Ask user to approve before proceeding to Phase 2

**### Phase 2: Generate Discussion**

Step 1 — Prepare:
- Load results-and-discussion.md for evidence reporting language
- Load conclusions-and-claims.md for calibrated interpretation language (suggests, indicates scope)
- Load anti-ai-patterns/vocabulary.md to screen generated output
- If user provided prior work to connect to, hold those references for connection sentences

Step 2 — Write Discussion Paragraphs:
- One paragraph per confirmed finding
- Each paragraph structure:
  1. Evidence sentence: state the finding with full quantification (use results-and-discussion.md patterns)
  2. Interpretation sentence: claim using calibrated language (use conclusions-and-claims.md: "suggests", "indicates")
  3. Literature connection sentence: if user provided prior work, connect; if not, write [CONNECT TO: describe the prior finding here]
- CRITICAL: interpretation sentence must follow evidence sentence — never lead with interpretation
- Anti-AI screen: after generating, check output against vocabulary.md; revise if flagged patterns appear

Step 3 — Output:
- Present all discussion paragraphs in sequence
- If file input, offer to append discussion to a file using Write tool
- Recommend Polish Skill for further expression refinement if user wants higher-register prose

**## Output Contract**
Table with outputs:
- pattern_analysis (Phase 1): structured Finding list (always)
- discussion_paragraphs (Phase 2): one paragraph per confirmed finding (after Phase 1 confirmation)
Include note: Phase 2 output cannot be produced without Phase 1 confirmation.

**## Edge Cases**
Table covering:
- Input is vague (no measurable values) → refuse Phase 1 with specific error message
- User skips Phase 1 confirmation and asks for discussion directly → require Phase 1 completion first
- User provides no research questions → ask once; if declined, write [RESEARCH QUESTION: describe your RQ here] placeholders
- User provides no prior literature → use [CONNECT TO: ...] placeholders; do not attempt to name papers
- Only one finding identified → produce single discussion paragraph; do not pad
- Finding interpretation conflicts with user-stated hypothesis → flag the discrepancy; do not suppress
- Journal specified but template missing → refuse with "Journal template for [X] not found. Available: CEUS."
- Input is LaTeX table markup → read data values and captions; ignore typesetting commands

**## Fallbacks**
Table covering:
- Structured Interaction unavailable → ask 1-3 plain-text questions (research questions + prior work + journal)
- Expression pattern leaf missing → proceed with general academic register; warn user of reduced quality
- Write tool fails → present discussion paragraphs in conversation
- Phase 1 produces no findings → report "No measurable findings identified from input" and stop

**## Examples**
One minimal invocation: user pastes a results table with two methods compared on accuracy. Show Phase 1 finding list output, then after user confirms, show one Phase 2 discussion paragraph with evidence → interpretation → [CONNECT TO: ...] structure.
