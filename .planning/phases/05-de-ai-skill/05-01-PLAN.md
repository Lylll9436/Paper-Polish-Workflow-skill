**Purpose:** One paragraph stating the Skill detects AI-generated patterns and rewrites them with explainable, risk-tagged results.

**Trigger:** Activates when user asks to reduce AI traces, de-AI, check for AI patterns, etc. Include Chinese triggers.

**Modes:**
| Mode | Default | Behavior |
- `direct` (default): Full detect-then-rewrite two-phase workflow with batch selection
- `batch`: Same operation across multiple files
- No `interactive` or `guided` modes -- this Skill always follows detect-then-rewrite

Mode inference: "scan only" / "just check" triggers detect-only (skip rewrite). Default includes both phases.

**References section:**
- Required (always loaded): anti-ai-patterns.md (overview with risk model), expression-patterns.md (for rewrite quality)
- Leaf hints (always loaded proactively): vocabulary.md, sentence-patterns.md, transitions-and-tone.md -- all three loaded at start for full-text scan
- Leaf hints (loaded during rewrite): expression pattern leaves based on detected section type
- Journal template: loaded when user specifies target journal; missing template triggers refusal per Phase 3/4 decision

**Ask Strategy:** Before starting, ask about: (1) target journal if not specified, (2) scope (full paper or specific section). In direct mode, skip if user provided enough context. Never ask more than 2 questions.

**Workflow -- Two-Phase Design (CRITICAL -- locked decision):**

Phase 1: Detect
1. Load all three anti-AI leaf modules proactively
2. If target journal specified, load journal template; if missing, refuse
3. Read user input (file via Read tool, or pasted text)
4. Scan full text against all three pattern dimensions (vocabulary, sentence-patterns, transitions-and-tone)
5. For each match: check domain term protection -- if the flagged term is standard domain terminology used in context (e.g., "landscape" in a geography paper), mark as SKIPPED with reason
6. For Optional-tier matches: only flag if the pattern appears 3+ times in the text (single occurrences suppressed to reduce noise)
7. Present detection summary: "Found N AI patterns (X High Risk / Y Medium Risk / Z Optional)" plus count of skipped domain terms
8. Present detailed list grouped by risk level (High first, then Medium, then Optional), each item showing:
   - Item number, risk level, pattern category (Vocabulary Inflation / Sentence Pattern / Transition & Tone)
   - Original text snippet (the sentence or clause containing the pattern)
   - Pattern explanation (which specific pattern from anti-AI library was matched and why)
   - Suggested replacement (from anti-AI library's replacement column)
9. Ask user which items to rewrite: "Fix all High Risk" / "Fix High + Medium" / "Fix all" / specific items by number

Phase 2: Rewrite
1. For selected items, group by paragraph -- when multiple items appear in the same paragraph, rewrite them together in one pass to maintain coherence
2. Load relevant expression pattern leaves based on section type for higher quality alternatives beyond the anti-AI replacement column
3. Rewrite flagged passages: restructure expressions, do not just swap synonyms. Preserve academic meaning and quality
4. If target journal specified, consider journal style preferences during rewrite
5. For file input: use Edit tool for in-place modifications, add `% [De-AI] Original: <original text>` LaTeX comment above each rewritten passage. Multi-line originals get one prefix per line. If existing `% [De-AI] Original:` annotations found, clean up before adding new ones
6. For pasted text: present rewritten version in conversation with before/after comparison
7. Generate rewrite report:
   - Total rewrites applied vs total detected
   - Breakdown by category (vocabulary inflation, sentence overclaim, transition smoothing) with counts and examples
   - Skipped items (Optional below threshold, domain terms protected)

**Output Contract:**
| Output | Format | Condition |
- Detection report | Structured markdown | Always (Phase 1) |
- Rewritten text | In-place LaTeX with annotations OR conversation output | Phase 2, depends on input mode |
- Rewrite report | Structured markdown table | After Phase 2 completes |
- Word count delta | Integer | After Phase 2 |

**Edge Cases:**
| Situation | Handling |
- No AI patterns detected | Report "No AI patterns detected" and exit; do not proceed to rewrite phase |
- Input too short (< 3 sentences) | Warn that detection may be unreliable on short text; proceed if user confirms |
- All detections are domain terms (all skipped) | Report "N patterns matched but all identified as domain terminology -- no rewrites needed" |
- User provides LaTeX with existing `% [De-AI] Original:` annotations | Clean up old annotations before running new scan |
- Input language is not English | Warn and suggest Translation Skill first |
- Journal template missing when journal specified | Refuse with message: "Journal template for [X] not found. Available: CEUS." per Phase 3/4 decision |

**Fallbacks:**
| Scenario | Fallback |
- Structured interaction unavailable | Ask 1-2 plain-text questions (journal + scope) |
- Reference file missing | Warn user, proceed with available modules (reduced detection coverage) |
- Target journal not specified | Ask once; if declined, use general academic style for rewrites |
