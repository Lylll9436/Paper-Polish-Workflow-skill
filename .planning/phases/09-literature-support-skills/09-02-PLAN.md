**## Purpose**
One paragraph: Skill generates complete, submission-ready academic cover letters by loading the target journal's template to align the contribution statement with journal scope and aims, collecting correspondence author details, and reading the paper (file or pasted text) to extract the paper title, key contribution, and data availability information. Output is a full letter with all four required content blocks: contribution statement (journal-aligned), data/code availability statement, conflict of interest declaration, and correspondence author contact. If the journal template is not found, the Skill refuses — it never falls back to generic academic prose, consistent with the journal-aware pattern used throughout this project.

**## Trigger**
Activates when user asks to write a cover letter, generate a submission letter, or draft an author letter for a paper submission. Include 6 bilingual examples (English + Chinese) covering: write a cover letter for a specific journal, generate a submission letter, draft an author letter, 帮我写投稿信, 帮我生成cover letter, 写一封投稿到[期刊]的信.

**## Modes**
Table with two rows:
- direct (default): single-pass letter generation; all context collected upfront via Ask Strategy
- batch: NOT supported — each letter requires specific paper context and author details

**## References**
Required table: none.
Leaf hints:
- references/journals/ceus.md: loaded when user targets CEUS; provides Aims & Scope, scope keywords for contribution alignment

Conditional: when journal specified, load references/journals/[journal].md. If template not found: refuse with "Journal template for [X] not found. Available: CEUS."

**## Ask Strategy**
The Skill needs four inputs. If already provided in trigger or paper content, skip the corresponding question:
1. Target journal: ask if not specified in trigger (required — determines which template to load)
2. Paper file or text: ask for file path or paste of the paper/abstract if not provided (required — for paper title and contribution extraction)
3. Correspondence author details: name, email, institution (required — ask as a single grouped question if not provided)
4. Data/code availability: are datasets or code publicly available? If yes, ask for repository URLs (optional — default to "Data are not publicly available" if user declines or does not know)

Ask at most 3 questions before proceeding. If correspondence details are missing but user declines: use [Corresponding Author Name], [Email], [Institution] placeholders.

**## Workflow**
Four steps:

Step 1 — Collect Context:
- Run Ask Strategy (journal, paper, author details, data availability) for any missing inputs
- Load references/journals/[journal].md — if not found: refuse with "Journal template for [X] not found. Available: CEUS." Do NOT proceed.
- If paper provided as file: use Read tool to load paper; extract title and key contribution
- If paper provided as pasted text: extract title and key contribution from the pasted content
- Extract Aims & Scope section from the loaded journal template (the contribution statement will reference this directly)

Step 2 — Draft Letter:
- Use the locked letter format from the interfaces block above
- Contribution statement: write 1–2 sentences explicitly referencing the journal's stated scope and aims from the loaded template. Do not use generic academic framing ("this paper advances the field of...") — use journal-specific framing ("This paper addresses [X], which aligns directly with [Journal]'s focus on [scope phrase from template]").
- Data/code availability: if repository URLs provided, list them; otherwise state "Data are not publicly available due to [reason if available, otherwise omit reason clause]."
- Conflict of interest: use standard declaration "The authors declare no conflict of interest."
- Contact block: use provided author details or placeholders [Name], [Email], [Institution] if not provided.

Step 3 — Output:
- File input: write to {input_filename_without_ext}_cover_letter.md using the Write tool
- Pasted text input: display the complete letter in conversation
- Confirm output location to user

**## Output Contract**
Table with output types:
- cover_letter_md: complete cover letter written to {input_filename_without_ext}_cover_letter.md (file input)
- cover_letter_text: complete cover letter displayed in conversation (pasted text input or Write tool failure)

Include a compact example showing the four required content blocks in order with their headings (**Contribution Statement**, **Data and Code Availability**, **Conflict of Interest**, **Corresponding Author**). Note: "Contribution statement must reference journal scope explicitly — not generic academic praise."

**## Edge Cases**
Table covering:
- Journal template not found → refuse: "Journal template for [X] not found. Available: CEUS." Do not generate with generic framing.
- Paper file not found or unreadable → ask user to paste paper text or abstract instead
- Correspondence author details not provided and user declines to supply → use bracketed placeholders; note that user must fill them before submission
- Data availability unknown → use "Data availability will be confirmed at revision stage." placeholder
- User specifies multiple authors for correspondence → ask which one is the corresponding author; list only one in the contact block
- Journal template found but Aims & Scope section is thin (few keywords) → use all available scope language from template; note contribution alignment may be approximate

**## Fallbacks**
Table covering:
- Structured Interaction unavailable → ask journal name and author details as sequential plain-text questions
- Read tool fails on paper file → ask user to paste abstract or key contribution sentences
- Write tool fails → display complete letter in conversation; advise user to copy to a .md file
- references/journals/ceus.md missing → refuse with journal-missing message; do not generate without template

**## Examples**
One minimal invocation example:
- User: "Write a cover letter for my CEUS submission" (with paper file path)
- Show: Skill loads ceus.md, extracts aims, reads paper, asks for author details, generates letter with all four blocks
- Show first 8 lines of the output letter including the contribution statement line referencing CEUS scope

Line budget: aim for 220–260 lines. The letter format example is the largest block — keep it as a compact illustration, not a full-length letter.
