Write all required sections per skill-conventions.md:

**## Purpose**
One paragraph: Skill generates or optimizes abstracts using the locked 5-sentence Farquhar formula. Both generate-from-scratch and restructure-existing paths produce labeled output followed by a clean version.

**## Trigger**
Activates when user asks to write, generate, optimize, or restructure an abstract. Include bilingual examples (English + Chinese).

**## Modes**
Table with two rows:
- direct (default): single-pass abstract output
- batch: NOT supported — abstract requires full paper context

**## References**
Required table: references/expression-patterns.md (always).
Leaf hints table: introduction-and-gap.md (contribution framing, sentences 1-2), results-and-discussion.md (evidence and result framing, sentences 4-5), conclusions-and-claims.md (calibrated claim language), methods-and-data.md (method description, sentence 3 if weak).
Conditional: when user specifies a journal, load references/journals/[journal].md; if missing, refuse with "Journal template for [X] not found. Available: CEUS."

**## Ask Strategy**
Ask before starting:
1. Path: does user have an existing abstract to restructure, or raw content to synthesize from? (skip if clear from input)
2. Target journal if not specified (ask once; if declined, use general academic style)
3. Word limit if known (typically 150-300 words)
Rules: never ask more than 2 questions before starting; skip path question if inference is clear.

**## Workflow**
Structure as three steps:

Step 1 — Collect Context:
- Load expression-patterns.md overview
- If journal specified, load journal template (refuse if missing)
- Read input (file via Read or pasted text)
- Determine path: restructure if input reads like a formed abstract; generate if raw materials. Ask if ambiguous.

Step 2a — Generate Path (raw content provided):
- Load introduction-and-gap.md (contribution framing)
- Load conclusions-and-claims.md (calibrated claim language)
- Load methods-and-data.md if method description is weak
- Synthesize 5 sentences following the locked formula
- Load results-and-discussion.md for sentences 4-5 (evidence + key result language)

Step 2b — Restructure Path (existing abstract provided):
- Load the same leaf hints as generate path
- Map each existing sentence to one formula position
- Rewrite to tighten formula compliance
- Run internal audit: confirm all 5 positions are filled
- Flag any missing positions with [MISSING: position-name] rather than silently omitting

Step 3 — Output:
- Present labeled version first (formula positions explicit)
- Present clean version (plain paragraph, no labels) separated by ---
- If user wants to save: offer to write to file using Write tool
- Optionally recommend Polish Skill for further expression refinement

**## Output Contract**
Table with two output rows: labeled_abstract (always), clean_abstract (always). Include example of the exact labeled format:
```
[1: Contribution] We propose...
[2: Difficulty] Despite growing interest in X, existing approaches fail to...
[3: Method] We address this by...
[4: Evidence] Evaluated on N datasets...
[5: Key Result] Our approach achieves...

---
Clean version:
We propose... Despite growing interest...
```

**## Edge Cases**
Table covering:
- Input is ambiguous (not clearly raw content or existing abstract) → ask before proceeding
- Existing abstract has fewer than 5 sentences → run restructure path; flag missing positions
- User specifies word limit → count words in output; warn if over limit
- Journal specified but template missing → refuse with "Journal template for [X] not found. Available: CEUS."
- Formula position cannot be filled from available content → flag with [MISSING: ...], do not invent content

**## Fallbacks**
Table covering:
- Structured Interaction unavailable → ask 1-2 plain-text questions (path + journal)
- Expression pattern leaf missing → proceed with general academic register; warn user
- Journal template missing (no journal specified) → ask once; if declined, use general academic style
- Write tool fails → present output in conversation

**## Examples**
One minimal invocation showing the labeled + clean output format.

Line budget: aim for 220-260 lines. Do not inline the full expression pattern tables — those live in the leaf files. Load at runtime.
