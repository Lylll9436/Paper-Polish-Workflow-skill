**## Purpose**
One paragraph: Skill generates or optimizes LaTeX figure/table captions for academic papers. Geography-aware: for spatial figures (maps, GIS outputs, photos), the Skill proactively collects study area, data source, and CRS metadata. Output is written directly to the user's .tex file at the correct \caption{} location. Captions follow target journal length and style when a journal is specified.

**## Trigger**
Activates when user asks to write, generate, optimize, or improve a figure or table caption. Include bilingual examples (English + Chinese). Six example invocations covering: write caption for figure, optimize table caption, generate CEUS-ready caption, 帮我写图题, 优化表格说明, 帮我生成符合期刊要求的图题.

**## Modes**
Table with two rows:
- direct (default): single-pass caption output, minimal interaction after Ask Strategy
- batch: NOT supported — each caption requires specific figure/table context

**## References**
Required table: references/expression-patterns.md (always — overview entrypoint).
Leaf hints table:
- references/expression-patterns/geography-domain.md: loaded for spatial figures (maps, GIS outputs, photos); provides study area framing, spatial description patterns, planning language
Conditional: when journal specified, load references/journals/[journal].md; if template not found, refuse with "Journal template for [X] not found. Available: CEUS."

**## Ask Strategy**
Write the geography-conditional Ask Strategy from the interfaces block above. The section must make explicit:
- Question 1 (always): Figure/table type (map / chart / diagram / photo / data table)
- Question 2 (always): Target journal if not specified
- Spatial branch (only if map/photo/GIS): study area, data source, CRS (optional), legend items (optional)
- Non-spatial branch (bar chart, line chart, schematic): skip geography; ask data source only if not mentioned
- Rule: never more than 3 questions before proceeding
- Rule: missing spatial metadata = skip the clause; do NOT insert [MISSING: ...] stubs
- Rule: if .tex file has multiple \caption{} commands, ask which label (e.g., \label{fig:map}) to target

**## Workflow**
Four steps:

Step 1 — Collect Context:
- Load references/expression-patterns.md overview
- If journal specified, load journal template (refuse if missing)
- Read input: if file provided, use Read tool; otherwise accept pasted text description
- Run Ask Strategy questions if any are still unanswered after reading input
- Determine path: if user provides an existing caption to improve → Optimize path; if user provides content description only → Generate path; ask if ambiguous

Step 2a — Generate Path (new caption from content description):
- If figure type is spatial: load references/expression-patterns/geography-domain.md
- Apply figure vs. table branching logic from interfaces block
- Compose caption using locked LaTeX format: \caption{[subject/area]. [content]. [data source].}
- Adapt length and style to journal template if loaded

Step 2b — Optimize Path (improve existing weak caption):
- If figure type is spatial: load references/expression-patterns/geography-domain.md
- Review existing caption for: completeness (area/content/source), spatial metadata gaps, terminology precision, journal length compliance
- Rewrite using the same branching and format rules as Generate path

Step 3 — Write to File:
- Read the target .tex file (required before Write — locate \caption{} insertion point)
- Replace the existing \caption{} content (or insert if empty) at the identified location
- Write updated file
- If Write tool fails: present final caption in conversation with instructions to paste manually

**## Output Contract**
Table with outputs:
- caption_latex: LaTeX \caption{...} string written to .tex file (always — file input)
- caption_latex: LaTeX \caption{...} string in conversation (fallback — Write fails, or pasted text input)
Include a concrete example showing the three-part structure:
- Spatial: \caption{Distribution of urban green space in Shenzhen Nanshan District. Data derived from OpenStreetMap 2023. Projected in WGS 84 / UTM Zone 50N.}
- Non-spatial: \caption{Comparison of model accuracy across three benchmark datasets. All values are F1 scores. Data from the COCO 2017 validation set.}
- Table: \caption{Descriptive statistics of land use change by category, 2010–2020. All areas reported in km². Data from the Third National Land Survey.}

**## Edge Cases**
Table covering:
- User specifies journal but template file not found → refuse: "Journal template for [X] not found. Available: CEUS."
- Figure type is spatial but no study area or data source provided → skip missing clauses; generate from available content; do NOT use [MISSING: ...]
- .tex file has multiple \caption{} commands → ask which \label{} is the target before Writing
- User provides both an existing caption and content description → default to Optimize path; confirm if ambiguous
- Figure type unclear (user says "my figure") → ask type before any branching logic
- Input is only a file path with no description → Read the file; if content is ambiguous, ask for clarification
- Non-standard caption format in journal (e.g., no data source required) → journal template overrides default format rules

**## Fallbacks**
Table covering:
- Structured Interaction unavailable → ask 1-3 plain-text questions (figure type, journal, study area if spatial)
- geography-domain.md leaf missing → proceed with general spatial description; warn user of reduced quality
- Journal template missing (no journal specified) → ask once; if declined, use general academic style
- Write tool fails → present caption in conversation; advise user to paste at \caption{} location
- .tex file not found or unreadable → present caption in conversation

**## Examples**
One minimal invocation: user provides description of a spatial map figure (urban heat island, Guangzhou, Landsat-8, 2022). Show Ask Strategy exchange (type: map, study area: Guangzhou, data source: Landsat-8), then the generated caption in LaTeX format. Show that it is written to the .tex file.

Line budget: aim for 220-260 lines. Do not inline geography pattern tables — those live in geography-domain.md and are loaded at runtime.
