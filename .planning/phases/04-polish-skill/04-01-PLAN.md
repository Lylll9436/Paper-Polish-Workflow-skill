- direct (Quick-fix) = default: intelligent single-pass, minimal interaction

- batch: Quick-fix across multiple sections/files

**Workflow** -- Two workflow paths clearly distinguished:

*Quick-fix (direct) Workflow:*
- Step 1 Collect Context: determine input type (file vs pasted), load references (expression patterns by section, anti-AI patterns proactively, journal template if specified), detect input characteristics (translationese, section type, length for smart adaptation)
- Step 2 Polish: single intelligent pass covering expression, logic, and structure as needed based on text quality; prioritize expression issues; apply journal style; avoid anti-AI vocabulary; pay special attention to translationese if detected; for file input use Edit tool with `% [Polish] Original: <original text>` annotation before each modification; for pasted text present polished output directly in conversation
- Step 3 Smart Adaptation: if input is long (4+ paragraphs or multiple \section{} markers), split by section and process each sequentially; maintain terminology consistency across sections
- Step 4 Summary: generate concise report with change count, modification types (expression/logic/structure), notes (translationese detected, etc.); include "Recommend running De-AI Skill for further detection check" when polishing made substantial rewrites

*Guided Mode Workflow:*
- Step 1 Collect Context: same as Quick-fix Step 1, plus confirm scope with user
- Step 2 Structure Pass: analyze paragraph organization, topic sentences, section flow, redundancy; present numbered checklist of problems found; user selects which to fix ("fix 1, 3" or "all"); apply selected fixes in batch using Edit tool with annotations; report changes
- Step 3 Logic Pass: re-read the current file (after structure edits); analyze argument coherence, claim support, terminology consistency; when multi-section input, include cross-section coherence check (argument chains, term consistency across sections); present checklist; user selects; apply fixes; report
- Step 4 Expression Pass: re-read the current file (after logic edits); analyze word choice, sentence clarity, academic tone, conciseness; load anti-AI patterns and flag AI-sounding phrases; detect and flag translationese if present; present checklist; user selects; apply fixes; report
- Step 5 Summary: same as Quick-fix Step 4

**LaTeX Annotation Format** -- Describe the annotation convention:
- Format: `% [Polish] Original: <original text>` on the line immediately before the replacement text
- Multi-line originals: each line gets its own `% [Polish] Original:` prefix
- Cleanup: after user confirms acceptance, remove all lines matching `^% \[Polish\] Original:` pattern
- Annotations are valid LaTeX comments -- file still compiles with them present

**Output Contract** -- Table:
- polished_text: always (in-place edits for file, conversation output for pasted text)
- change_annotations: for file input only (LaTeX comments preserving originals)
- summary_report: always (in conversation, not in file)

**Edge Cases** -- Table covering at minimum:
- Input too short (single sentence) -> polish but warn limited context for structural assessment
- No section headers -> default to expression-focused polishing
- Pasted text input -> output directly in conversation, no file operations
- File input -> use Edit tool only, never Write (do not overwrite entire file)
- Journal specified but no template -> refuse, instruct user to add template (per locked decision)
- Very long input (10+ sections) -> process in batches, maintain cross-section awareness
- Text with existing `% [Polish] Original:` annotations -> clean up old annotations before adding new ones (avoid nesting)
- Mixed LaTeX and text -> preserve all LaTeX commands, polish surrounding text only

**Fallbacks** -- Table:
- Structured interaction unavailable -> plain-text questions (1-3 max)
- Expression pattern leaf missing -> use overview entrypoint
- Journal template missing -> refuse per locked decision (NOT proceed with general style)
- Anti-AI patterns file missing -> proceed without, rely on Claude's own vocabulary awareness

**Line budget compliance:** Target 250-290 lines. The Polish Skill has more concerns than Translation (two modes, three guided steps, annotation format). Manage this by:
- Describing the guided step pattern ONCE then using a compact table for step-specific focus areas
- Keeping Quick-fix workflow brief (3-4 bullets per step)
- Using tables over prose wherever possible
- NOT inlining any reference content
